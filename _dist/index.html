Elite Receipt was my first project with the Innovations team. It was a Walmart sponsored, receipt scan to rewards mobile web application that was created as a demo for a Hot Wheels test campaign. It interfaced through a custom admin page to handle campaign data and user data to maintain their scanned receipts, their points, and rewards.

The Challenge: The program developed by a contractor no longer with the company. I assessed the application and saw that it lacked any sustainable architecture in the JS, HTML, and the CSS. Also, the Hot Wheels campaign was hard coded into the application. I was tasked with optimizing the mobile site for scalability, designing and building a desktop version of the site cross browser compatible down to IE7 (in 2 weeks), abstracting all styling and images for both mobile and desktop so we could implement custom branding, and creating a CMS interface within the admin site.

The Desktop: The most urgent was the desktop version. I made a quick mock to demonstrate the general design while keeping in mind that the styling will need to be abstracted so keeping it simple with style reusability. I created a prototype to keep track of all the possible pages and interaction between the pages of the application and presented those to the project manager. He approved the mock and prototype. I managed to complete it within the 2 weeks.

Styling and Campaign Abstraction: Because the CMS had not yet been implemented into the Admin site, I abstracted the styling and branding configuration into a folder using the ID for the campaign being used. The campaign is set by a data attribute on the script tag loading in the JS. This folder contained, SCSS, images and a JSON configuration file that manages all of the campaign information including environment variables. I developed the desktop application also using the MVC boilerplate I created.

Refactoring: Next task was to refactor the mobile version and abstract the styling. The mobile version was a single page app. The HTML page was huge containing all of the dynamic templates used within the HTML page. The CSS used no preprocessing engine. The JS contained 3 files each filled with global functions and containing (no joke) objects as large as 3000 lines of code. I was allowed 3 days for the initial refactoring and 1 week for the styling and campaign abstraction.

First, I put in place Grunt because that would be the foundation for refactoring to be able to run Grunt tasks. I changed all of the CSS to SCSS files and broke them up into smaller more logical chunks. Ran them through Grunt to minify and concatenate them to single file with maps for easier development troubleshooting.

I added RequireJS and used Jquery extend to break up the the massive JS files and objects into smaller files. By using RequireJS and Jquery extend I could take a single object, break it up into as many logical files as I want and Jquery extend would pull them all together into a single object. RequireJS with Grunt would then allow me to minify and concatenate those JS so there would be no issue with race conditions having those files separated out. It was a very time efficient, non-intrusive solution.

I removed the HTML templates from the main HTML file and put them each in a separate file and formatted them for doT templates. With that I am able to compile with Grunt, the templates into a JS file and have that JS file minified and concatenated with the other JS so it would have little network impact.

From JSON to CMS: Lastly, I had to take the configurations and styling from the JSON and implement those into the CMS. I did this a little bit at a time but in the end the JSON file was left only a few configuration variables and designers could then use the CMS to style the next campaign.

This project went on to support campaigns for Hot Wheels, Crystal Light, Unilever (Suave, Tresemme, Axe), Walgreens, Walmart, and Kraft

Blue was a pet project of mine as a solution for jump starting my projects. I call it Blue, as in 'Blue Collar', a simple, layman's approach to a framework that just helps you organize and engineer large scale or small applications from the ground up using a vanilla JS OOP style approach.

A Little History: When I was developing in Flash AS3 I didn't know of any project frameworks (before Flex). I developed my own AS3 MVC framework which gave me a foundation for all my projects. The projects were extremely varied from presentation, training tool applications to games. Having this framework constantly evolving as I used it throughout my Flash career for about 5+ years, this gave me a solid understanding for what was required and how to develop a framework.

Why I built it? When I first got into JavaScript, I had jumped in with both feet into a heavily customized Backbone boilerplate project with Walmart that would run for about 3 months. I looked into other frameworks like Knockout and Angular. Afterward in retrospect I took what I liked and what I didn't like. I worked in an environment with all levels of JS developers. Backbone and other frameworks seemed to make JavaScript not look like JavaScript anymore, and some even violated web standards. It was my opinion if you threw a new developer in the world of Backbone, Angular or many of the other frameworks, and then threw them into a regular JS project they wouldn't know what to do.

Success: This framework was a successful attempt at creating a framework that that could be used for any project of any size and to be maintainable and understood by developers of all levels providing they had an understanding of vanilla JS OOP using prototype and closures. It would also help the new JavaScript developer learn the ins and outs of a simple framework and relationships with their components. While learning best practices, coding habits, and skills that could be assimilated into any framework or used in developing an application from the ground up.

During my 3 years with Lunchbox I used this framework for nearly all my projects and also used it for mentoring. A few of the more successful applications created using Blue were Elite Receipt and Firestarter.


Features:
  • Built on JQuery, RequireJS, doT, Validate, Grunt, SASS, and Bootstrap (optional)
  • Flexible for multi-page or pageless designs using routes through custom hash management.
  • JSON configurations are loaded into the model. An environment variable is used to compare against the url to determine your environment specific properties like API service URLs. It is also used for custom environment behaviors like disabling console log in production.
  • Ability to work using minified or un-minified JS for faster development and troubleshooting.
  • Query strings are used for debug modes.
  • Form handling and validation using Validate, including inline validation capability when input is off focus.
  • Modal handling using dynamic templates.
  • The custom build script calling Grunt tasks offer these options:
    • Min/concat JS and templates and creates source maps for debugging
    • Cache busting for js, json, and css files
    • Compile doT templates to JS
    • Compress PNG
    • Create spritesheets and generate SASS files
    • Watch and compile SCSS
